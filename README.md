## 프로젝트 소개

IgLoo는 기존의 당근마켓, 번개장터, 중고나라와 같은 **중고거래 서비스 모델에 ‘경매 낙찰 시스템’을 도입**하여 거래의 재미와 효율성을 극대화한 **실시간 로컬 경매 서비스**입니다.

---

## 프로젝트 개요

### 프로젝트 정의

기존의 중고거래 서비스 모델에 **경매 낙찰 시스템**을 결합하여,  
**짧은 시간 안에 거래를 성사**시키고 **경매의 재미**를 제공하는 것을 목표로 합니다.

### 핵심 차별화 포인트 (vs eBay)

| 비교 항목   | 우리 서비스 (로컬 기반 실시간 경매)                       | 이베이 (eBay)                  |
| ----------- | --------------------------------------------------------- | ------------------------------ |
| 경매 기간   | **초단기 설정** (1시간 / 3시간 / 6시간 / 12시간 / 24시간) | 3일 ~ 10일                     |
| 거래 방식   | **로컬(직거래) + 택배** 두 가지 옵션 모두 제공            | 택배 거래 위주 고정            |
| 가격 제안   | **카테고리 기반 경매 가격 추천** 및 유사 상품 제안        | 일반적인 검색 및 리스팅        |
| 결제 안전성 | **에스크로 방식 결제 시스템**으로 안전성 강화             | 플랫폼 자체 보호 프로그램 기반 |

### 주요 기능 (Key Features)

- **실시간 경매 시스템:** 사용자 설정에 따른 초단기 타임 세일형 경매 진행
- **지능형 가격 추천:** 같은 카테고리 내 유사 상품 데이터를 기반으로 적정 입찰/낙찰가 가이드 제공
- **하이브리드 거래 모델:** 동네 기반의 직거래 편의성과 원거리 택배 거래의 확장성을 동시에 제공
- **안전 결제(Escrow):** 낙찰 후 대금을 플랫폼이 예치하고, 구매 확정 시 판매자에게 정산되는 안전한 프로세스 구축

### 기대 효과

- **판매자:** 경매 방식을 통해 정해진 시간 내에 빠른 물품 처분 및 최적의 가격 방어 가능
- **구매자:** 실시간 입찰을 통한 게임화(Gamification) 요소와 에스크로를 통한 사기 예방 신뢰도 확보

## 사용 기술

### BE

- **Framework**
  - Java 21
  - Spring Boot 3.5.9
- **Data**
  - Spring Data JPA
  - Querydsl
  - MySQL
  - Redis - Caching & Redisson Lock을 이용하여 실시간 경매 중복 입찰 시 데이터 꼬임 방지
- **Real-Time**
  - STOMP (WebSocket)
- **Batch**
  - Reader - Processor - Writer 순으로 경매 완료된 경매품들 상태 변경
- **Auth**
  - Spring Security
  - JWT
  - OAuth2
- **Infra**
  - AWS
    - EC2
    - S3
    - RDS
  - Docker
  - GitHub Actions - CI/CD

### Monitoring

- Grafana (목표)
